      # git问题集锦

      # 1.查看远程信息库

      • 当你从远程仓库克隆时，实际上Git自动把本地的master分支和远程的master分支对应起来了，并且远程仓库的默认名称是origin。要查看远程库的信息用git remote
      $ git remote ---输出origin
或者用git remote -v显示更详细的信息
$ git remote -v
---输出
origin http://10.10.10.10:8080/项目目录/abc-abc-web.git (fetch)
origin http://10.10.10.10:8080/项目目录/abc-abc-web.git (push)

      # 2.下载一个新项目

      • 下载一个新项目并在本地新建开发分支，拉去远程开发分支代码到本地
      1. git clone http://10.10.10.10:8080/项目目录/abc-abc-web.git ---克隆项目
2. git fetch ---拉取所有分支
3. git checkout -b dev origin/dev ---切换到dev分支
4. git pull origin dev ---将dev分支拉取到本地dev分支

      # 3.git 打开忽略识别文件夹大小写命令

      git config core.ignorecase false

      # 4.git push origin与git push -u origin master的区别

      $ git push origin命令表示，将当前分支推送到origin主机的对应分支。 

如果当前分支只有一个追踪分支，那么主机名都可以省略。 

$ git push 如果当前分支与多个主机存在追踪关系，那么这个时候-u选项会指定一个默认主机，这样后面就可以不加任何参数使用git push。

$ git push -u origin master 上面命令将本地的master分支推送到origin主机，同时指定origin为默认主机，后面就可以不加任何参数使用git push了。

不带任何参数的git push，默认只推送当前分支，这叫做simple方式。此外，还有一种matching方式，会推送所有有对应的远程分支的本地分支。Git 2.0版本之前，默认采用matching方法，现在改为默认采用simple方式。
